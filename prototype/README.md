# Overthrone Single-Player Prototype

Unity 프로젝트가 생성되기 전, GDD의 핵심 루프를 빠르게 검증하기 위한 브라우저 기반 Three.js 3D 프로토타입입니다.

## 실행

```bash
npm install
npm run dev
```

브라우저에서 `http://127.0.0.1:5173`을 엽니다.

## 현재 구현 범위

- 메인 화면
- 방 생성 화면
- 진영 선택
- AI 플레이어 수 선택
- 게임 시작
- 달빛 정원 스타일의 테스트 맵
- 플레이어 이동과 마우스 시점
- 3인칭 추격 카메라
- 슬라임 스킬
- 점령 포인트 3개
- 팀별 목표
- 점령에 따른 공격자/왕 상태 변화
- AI 플레이어 이동, 점령, 추격, 붙들기, 포획

## 조작

| 입력 | 행동 |
| --- | --- |
| `WASD` | 이동 |
| `클릭 1회` | 마우스 시점 조작 시작 |
| `Mouse` | 시점 회전 |
| `Shift` | 스프린트 |
| `R` | 슬라임 스킬 |
| `E` / 마우스 좌클릭 | 덮치기 |
| `F` | 왕 상태에서 붙들린 대상 포획 |
| `Tab` | 점수판 |
| `Esc` | 마우스 잠금 해제 |

## 상태별 이동 정책

| 상태 | 이동 | 스프린트 | 속도 |
| --- | --- | --- | --- |
| 중립 | 가능 | 가능 | 기본 |
| 공격자 | 가능 | 가능 | 기본보다 약간 빠름 |
| 왕 | 가능 | 가능 | 공격자보다 빠름 |
| 붙들림 | 불가 | 불가 | 정지 |
| 포획됨 | 불가 | 불가 | 정지 |

추가로 슬라임 상태에서는 이동 속도가 증가하고, 붙든 대상이 있으면 포획 행동만 가능하며 이동은 멈춥니다.

## 설계 메모

이 프로토타입은 최종 엔진 결정을 대체하지 않습니다. 규칙 검증용이며, Unity 전환 시 `prototype/src/game.js`의 상태 전환 규칙을 기준으로 C# 시스템으로 이식하는 것이 목표입니다.
